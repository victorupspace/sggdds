# Accessibility

Todo componente futuro deve ser projetado, implementado e revisado com referencia nas WCAG aplicaveis ao produto.

## Criterios minimos

- Semantica HTML correta antes de adicionar ARIA.
- Navegacao por teclado completa e previsivel.
- Estados de foco visiveis.
- Contraste adequado para texto, icones funcionais e estados.
- Labels, descricoes e mensagens de erro programaticamente associadas.
- Responsividade sem perda de operabilidade em mobile web.
- Stories com estados relevantes para auditoria visual e funcional.

## Documentacao por componente

Cada componente deve documentar uso esperado, estados, comportamento com teclado, semantica HTML, responsividade, variacoes, criterios de acessibilidade e exemplos permitidos e nao permitidos.
